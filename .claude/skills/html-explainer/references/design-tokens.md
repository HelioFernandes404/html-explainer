## Color tokens

```css
:root {
  /* Backgrounds */
  --canvas:  #0d1117;   /* page background, main containers */
  --surface: #161b22;   /* table headers, secondary panels, code bg */

  /* Text */
  --fg:          #e6edf3;  /* primary text, headings, values */
  --fg-muted:    #7d8590;  /* labels, captions, metadata, secondary */
  --fg-inverse:  #ffffff;  /* text on primary/danger buttons */

  /* Accent */
  --accent:       #2f81f7;         /* links, input focus, interactive highlights */
  --accent-glow:  rgba(47,129,247,0.3); /* focus ring, input shadow */

  /* Borders */
  --border:       #30363d;  /* default: sections, tables, buttons, inputs */
  --border-hover: #8b949e;  /* hover state on interactive borders */

  /* Buttons */
  --btn-bg:       #21262d;  /* neutral button bg */
  --btn-hover:    #30363d;  /* neutral button hover bg */
  --success:      #238636;  /* primary / affirmative action bg */
  --success-hover:#2ea043;
  --danger:       #f85149;  /* destructive action text or icon */
  --danger-hover: #b62324;  /* destructive action hover bg */

  /* Code */
  --code-bg: rgba(110,118,129,0.4); /* inline code background */

  /* Table */
  --row-even: rgba(22,27,34,0.5);   /* alternating row tint */
}
```

## Typography

```css
:root {
  --mono: "JetBrainsMono Nerd Font", "JetBrains Mono", monospace;
  --sans: "Inter", system-ui, -apple-system, sans-serif;
}

body {
  font-family: var(--mono);  /* body text, headings, everything technical */
  font-size: 16px;
  font-weight: 400;
  line-height: 1.6;
}

/* UI controls only */
button, input, select, label {
  font-family: var(--sans);
  font-size: 14px;
  font-weight: 500;
  line-height: 1.4;
}
```

Note: using `--mono` for body text is intentional. It signals developer tooling, documentation, and technical content. Don't replace it with a generic sans-serif for paragraph text.

## Accessibility

Contrast ratios (dark background #0d1117):
- `--fg` (#e6edf3): **~16:1** — excellent
- `--fg-muted` (#7d8590): **~5.1:1** — AA compliant
- `--accent` (#2f81f7): **~5.1:1** — AA compliant
- white on `--success` (#238636): **~4.6:1** — AA compliant (borderline for small text)
- `--danger` (#f85149): **~5.7:1** — AA compliant

Always preserve focus styles. The blue glow (`--accent-glow`) is part of the visual identity, not just accessibility.

---

## Do / Don't

**Do:**
- Use `--border` for all dividers, table borders, button/input outlines
- Use `--surface` for secondary backgrounds (table headers, code blocks, panels)
- Use `--accent` only for links, focus, interactive highlights
- Use `--success` only for primary affirmative actions
- Use `--danger` only for destructive actions or error states
- Use `--mono` for all body text and headings

**Don't:**
- Introduce gradients for decoration
- Use heavy box shadows for depth
- Turn every section into a card — a border or divider is usually enough
- Use `--accent` as a large background color
- Remove focus indicators
- Load external fonts — system stacks are sufficient
